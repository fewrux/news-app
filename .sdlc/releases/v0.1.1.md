# Release v0.1.1 — Consolidate the SDLC operator surface around the DSL

Pure operator-surface consolidation. Makes `.sdlc/sdlc.yaml` the canonical
declaration of policy and trims the always-applied Cursor rule tier so the
surface is thin pointers into the DSL. No app-surface changes.

`merged_at` is intentionally empty — per `.cursor/rules/provenance.mdc` we
never fabricate values; the squash-merge commit timestamp on `main` is the
source of truth.

## Summary

- **Demoted three over-broad always-applied rules** to context-scoped
  activation (bodies unchanged; only the activation tier moves):
  - `provenance.mdc` → `globs: .sdlc/**/*.md` (fires when editing an SDLC
    artifact; merge still gated by
    `gate.review_approved.requires.provenance_present`).
  - `branch-discipline.mdc` → `globs: .github/workflows/**` (mechanical
    enforcement stays in `guard-shell.mjs`).
  - `commit-conventions.mdc` → agent-requested via a precise `description:`
    (fires when composing a commit).
- **Trimmed two always-applied rules to thin pointers**: `agent-autonomy.mdc`
  (30 lines, "overrides any upstream" sentence preserved verbatim, cites
  `policies.autonomy`) and `sdlc-loop.mdc` (20 lines, cites `AGENTS.md` and
  `sdlc.yaml`).
- **Slimmed `.cursor/hooks/load-context.mjs`** to dynamic-only content:
  removed the `Slash commands:` / `Free-tier only:` / `Autonomy:` lines
  (now covered by always-applied rules + `AGENTS.md`). Per the SPEC-0003
  AC-6 amendment this release also reduces the open-handoffs banner to
  **ids-only** (dropping per-entry `intent:/spec:/adrs:/tracker:` metadata,
  chiefly the tracker UUIDs) and adds a **non-optional first-reply
  directive** so the agent reliably surfaces the queue even when the
  opening message is unrelated.
- **De-duplicated** the repeated `sessionStart`-hook paragraph in
  `AGENTS.md`.
- **Retiered** `sdlc.yaml.instructions.rules.registry`: exactly three rules
  remain `always_apply: true` (`sdlc-loop`, `free-tier-only`,
  `agent-autonomy`).

Combined always-applied rule line count dropped from 386 to **78 lines
(≈ 80% reduction)** — the per-interactive-turn context tax this consolidation
targeted.

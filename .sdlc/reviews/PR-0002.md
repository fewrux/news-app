### Minor / nits (non-blocking)

1. **Commit `5bb2a2f` subject is 77 characters, exceeds the `≤ 72` hard rule.**
   _Cite: `.cursor/rules/commit-conventions.mdc` "Subject line ≤ 72 chars".
   Subject: `chore(sdlc): broaden SDLC discoverability across humans and non-Cursor agents`._
   Non-blocking because the PR will land via squash-merge, and the maintainer
   can shorten the squash subject at merge time (e.g.
   `chore(sdlc): broaden SDLC discoverability for non-Cursor agents` = 65 chars).
   Recommend tightening the squash subject before merge.

2. **`INT-0001-sdlc-discoverability.md` has `created_at` at the top level of
   the frontmatter rather than inside the `provenance:` block.**
   _Cite: `sdlc.yaml.artifacts.common_provenance.required_fields` and
   `.cursor/rules/provenance.mdc` (`created_at: <ISO-8601 UTC>` listed under
   `provenance:`)._
   The value (`2026-05-25T19:22:50Z`) is present in the frontmatter and is
   honest, so the spirit of the contract is met; the structural location
   is one indentation level off. Recommend moving it inside `provenance:`
   in a follow-up commit (this PR has already shipped the rest of the
   value; a single-line frontmatter fix is the cheapest remediation).

3. **`docs/.plane-pages.json` is committed in a transitional state.**
   The file contains 18 `_zombie_run1_*` / `_zombie_run2_*` keys (pre-cleanup
   pointers from earlier sync attempts) that do not correspond to any current
   file under `docs/`. The `syncDocs()` upsert is keyed by `<rootRel>/<filename>`,
   so the zombie entries are inert — they are neither matched against any
   current doc nor used to PATCH, only re-saved verbatim by `savePagesMap`.
   The local working tree already carries a cleaned-up version of this file
   (keys re-shaped to `docs/<filename>`); committing that cleanup in a
   follow-up would remove the inert references and the 18 abandoned Plane
   pages they point to (use `purge-docs` first, then re-sync).
   _Cite: harmless drift; no rule cites apply — this is hygiene._

### Informational (no action required)

1. **`SKILL.md` for plane-sync claims "No state file is needed"; the
   implementation requires one.** The script comment explains why ("Plane's
   `/pages/` list endpoint does NOT expose `external_id` / `external_source`
   in the summary payload"), and the SKILL.md was updated in this PR to add
   the `purge-docs` command but the "no state file" line under the
   `sync-docs` bullet still reads as authoritative. The script's behavior
   wins on conflict, and the contradiction is small. Worth tightening in a
   `/learn` pass.
   _Cite: `.cursor/skills/plane-sync/SKILL.md` "sync-docs" bullet vs
   `scripts/plane-sync.mjs:288-303` ("we cannot rely on the API alone for
   idempotency without N+1 detail fetches")._

2. **Diff size 1949 / 37 = ~1986 LOC exceeds
   `sdlc.yaml.policies.autonomy.pause_on: "diff > 400 LOC without spec update"`.**
   Surfaced explicitly in the PR body with bundling rationale. Per
   `sdlc.yaml.workflows.chore.steps: [implement, verify, review]`, a chore
   workflow has no `spec` phase; `INT-0001` was back-filled as the closest
   equivalent. Acceptable as a one-off; consider whether large-chore PRs
   should default to a lightweight spec going forward
   (`/learn` candidate).

3. **`sdlc.yaml.instructions.hooks.registry` still names `.sh` scripts

   inconsistency (also flagged in `.sdlc/reviews/PR-0001.md` finding #4);
   not introduced by this PR. Tracking only — let the next PR that touches
   the hooks registry reconcile.

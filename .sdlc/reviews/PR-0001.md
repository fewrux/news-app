### Minor / nits (non-blocking)

1. **Status check name drift is declared but not yet reconciled.**
   `sdlc.yaml.integrations.github.branch_strategy.protection.require_status_checks`
   lists `[ci/lint, ci/typecheck, ci/build, ci/e2e]`, but actual job names
   are `lint, typecheck, build, e2e`, and `e2e-evidence.yml` is referenced in
   `integrations.github.actions.workflows` but does not exist in
   `.github/workflows/`. The PR description explicitly defers this as
   follow-up #1; acceptable. Tracking only.
   _Cite: `.sdlc/sdlc.yaml:556` and `.sdlc/sdlc.yaml:566`._
2. **`.gitignore` `.env*` is broader than necessary.** Matches any
   `.env.test.example` or `.env.production.example` someone might add later
   and silently un-tracks them. `.env.example` is already tracked so it's
   safe in the current state, but a narrower set (`.env`, `.env.local`,
   `.env.*.local`) would be more intent-revealing. Non-blocking.
   _Cite: `.gitignore:49`._
3. **PR scope creep is documented but real.** Title covers only the SDLC
   contract changes; the PR also lands `chore(gitignore)` and the
   `fix(plane-sync)` hot-fix. The body explains each commit and the
   plane-sync fix surfaced from PR #1's own CI, so the cohesion holds. Per
   `.cursor/rules/commit-conventions.mdc` "one logical change per commit",
   the four commits are individually compliant. Future PRs of this size
   should split or update the title to reflect all scopes.
4. **`sdlc.yaml.instructions.hooks.registry` still names `.sh` scripts**
   while `.cursor/hooks.json` correctly registers `.mjs`. Pre-existing
   inconsistency, not introduced by this PR. Out of scope; flag for
   /learn-phase cleanup.
   _Cite: `.sdlc/sdlc.yaml:501-504` vs `.cursor/hooks.json`._
5. **Hook `onMainBranch()` reads `.git/HEAD` directly.** Works for the
   canonical checkout; would silently return `false` in a worktree where
   `.git` is a file pointer. Acceptable for now (no worktrees in this
   workflow), but `git rev-parse --abbrev-ref HEAD` would be more robust.
   Edge-case nit.
